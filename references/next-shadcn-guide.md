# Next.js 15 + Tailwind CSS v4 + shadcn/ui 参考指南

## 什么是 shadcn/ui？

shadcn/ui **不是 npm 包**，而是一套**可复制源码的组件集合**：

- 底层基于 **Radix UI**（无障碍原语）+ **Tailwind CSS**（样式）
- 组件源码直接复制到你的项目（`src/components/ui/`）
- 你完全拥有代码，可以自由修改
- 40+ 现成组件，开箱即专业设计
- **Vercel 官方推荐**，Next.js 生态首选 UI 方案

## 快速开始

### Step 1: 创建 Next.js 项目

```bash
npx create-next-app@latest my-app \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --src-dir \
  --import-alias "@/*"

cd my-app
```

> **交互式选项**（推荐）：
> - TypeScript: **Yes**
> - ESLint: **Yes**
> - Tailwind CSS: **Yes**
> - `src/` directory: **Yes**
> - App Router: **Yes**（必须，shadcn/ui 依赖 App Router）
> - Import alias: **@/\***

---

### Step 2: 初始化 shadcn/ui

```bash
npx shadcn@latest init
```

CLI 会问你几个问题：

| 问题 | 推荐答案 | 说明 |
|------|---------|------|
| Which style would you like to use? | **New York** | 更现代的设计风格 |
| Which color would you like to use as base? | **Zinc** | 最流行的灰色基调 |
| Do you want to use CSS variables for colors? | **Yes** | 支持深色模式 |
| Are you using a custom `@tailwindcss/vite` plugin? | **No** | Next.js 内置支持 |
| Where is your global CSS file? | `src/app/globals.css` | |
| Where is your tailwind.config located? | `tailwind.config.ts` | v3 需此文件；v4 不需要 |
| Configure the import alias for components? | `@/components` | |
| Configure the import alias for utils? | `@/lib/utils` | |
| Are you using React 19? | **Yes** | Next.js 15 默认 React 19 |
| Would you like to use rsc? | **Yes** | Server Components 是默认推荐 |

---

### Step 3: 安装 shadcn/ui 组件

```bash
# 一次安装多个常用组件
npx shadcn@latest add button card dialog dropdown-menu input label select table tabs badge avatar

# 或单独安装
npx shadcn@latest add button
npx shadcn@latest add card
```

### Form（配合 React Hook Form）

```bash
npm install react-hook-form @hookform/resolvers zod
npx shadcn@latest add form
```

```tsx
import { zodResolver } from "@hookform/resolvers/zod"
import { useForm } from "react-hook-form"
import * as z from "zod"
import {
  Form,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from "@/components/ui/form"
import { Button } from "@/components/ui/button"
import { Input } from "@/components/ui/input"

const formSchema = z.object({
  username: z.string().min(2, "At least 2 characters"),
})

export function ProfileForm() {
  const form = useForm<z.infer<typeof formSchema>>({
    resolver: zodResolver(formSchema),
    defaultValues: { username: "" },
  })

  function onSubmit(values: z.infer<typeof formSchema>) {
    console.log(values)
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-8">
        <FormField
          control={form.control}
          name="username"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Username</FormLabel>
              <FormControl>
                <Input placeholder="shadcn" {...field} />
              </FormControl>
              <FormDescription>
                This is your public display name.
              </FormDescription>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type="submit">Submit</Button>
      </form>
    </Form>
  )
}
```

## Tailwind CSS v3 vs v4

### Tailwind v3（当前 create-next-app 默认）

```typescript
// tailwind.config.ts
import type { Config } from "tailwindcss"

const config: Config = {
  content: [
    "./src/components/**/*.{ts,tsx}",
    "./src/app/**/*.{ts,tsx}",
  ],
  theme: {
    extend: {
      colors: {
        primary: "#3b82f6",
      },
    },
  },
  plugins: [require("tailwindcss-animate")],
}

export default config
```

```css
/* src/app/globals.css */
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### Tailwind v4（未来方向）

```css
/* src/app/globals.css */
@import "tailwindcss";

@theme {
  --color-primary-500: #3b82f6;
}
```

```typescript
// next.config.ts — v4 无需 tailwind 插件
import type { NextConfig } from "next";

const nextConfig: NextConfig = {};
export default nextConfig;
```

> **当前推荐**：使用 Tailwind v3（`create-next-app` 默认），兼容性最好。
> Tailwind v4 稳定后可在 Next.js 中使用 `@tailwindcss/vite` 插件。

## 与 Radix UI 的关系

| | Radix UI | shadcn/ui |
|---|---|---|
| **定位** | 无头组件原语（Headless） | 完整设计系统 |
| **样式** | 无（需自己写 CSS） | Tailwind CSS（开箱即用） |
| **可定制性** | 高（完全自己控制样式） | 高（拥有源码，自由修改） |
| **使用方式** | npm 包（import） | 复制源码（不 import） |
| **推荐场景** | 需要完全自定义样式 | 快速开发、专业设计 |

**shadcn/ui = Radix UI + Tailwind CSS + 专业设计**

## Next.js 15 关键概念

### App Router（文件系统路由）

```
src/app/
├── layout.tsx          # 根布局（所有页面共享）
├── page.tsx            # / 首页
├── about/page.tsx      # /about
├── dashboard/
│   ├── page.tsx       # /dashboard
│   └── layout.tsx     # dashboard 专属布局
└── api/
    └── route.ts       # API Route（后端接口）
```

### React Server Components (RSC)

Next.js 15 默认所有组件都是 **Server Component**：

```tsx
// Server Component（默认）
// ❌ 不能使用 useState, useEffect
// ✅ 可以直接读数据库、调用 API

export default async function Page() {
  const data = await fetchData()  // 直接在服务器获取数据
  return <div>{data}</div>
}

// Client Component（需显式声明）
"use client"

import { useState } from "react"

export default function Counter() {
  const [count, setCount] = useState(0)  // ✅ 可以用 hooks
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```

### 数据获取

```tsx
// Server Component 中直接 fetch（推荐）
export default async function Page() {
  const res = await fetch("https://api.example.com/data", {
    cache: "no-store",  // 禁用缓存（每次都重新获取）
  })
  const data = await res.json()
  return <div>{JSON.stringify(data)}</div>
}
```

## 部署

### Vercel（推荐，一键部署）

```bash
# 1. 推送到 GitHub
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourname/my-app.git
git push -u origin main

# 2. 在 Vercel 导入仓库（自动检测 Next.js）
# 3. 每次 git push 自动部署
```

### 静态导出（不适合 App Router）

```bash
# ⚠️ App Router 不支持完整的静态导出
# 如果需要纯静态，使用 Pages Router 或导出为 Node.js 服务器
npm run build
npm start
```
